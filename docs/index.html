Hugging Face · Ollama,
in your terminal.

hfo is a fullscreen terminal UI that turns any Hugging Face repo URL into a running Ollama model — auto-tuned to your hardware. Discover, install, tune, back up, restore, and launch integrations (Claude Code, Codex, Copilot CLI, OpenCode, Droid, Cline, Hermes, Kimi, OpenClaw, Pi, VS Code) without leaving your shell.

What it does

Everything that normally takes a dozen shell commands and a hand-written Modelfile — folded into one TUI that understands your rig.

Fullscreen TUI

Tab-based shell with a live dashboard (GPU / VRAM / RAM meters, loaded models, tier score). Alt-screen buffer preserves your scrollback on exit.

Hardware-aware install

Scores every GGUF quant in a repo against your VRAM, parses the HF model card for recommended temperature / top_p / ctx, synthesizes a tuned Modelfile, and registers with ollama create.

Launch integrations

One-key L spawns any Ollama integration (Claude Code, Codex, Copilot CLI, OpenCode, Droid, Cline, Hermes, Kimi, OpenClaw, Pi, VS Code) using the selected model as the local backend.

Backup · Restore

Zip the model folder with level-9 compression, write a sidecar metadata.json, then restore with one command — regenerating the Modelfile if it's missing.

Orphan reinstall

Models removed from Ollama but still on disk surface as "available to reinstall". I rebuilds the Modelfile from the GGUF alone using your hardware + HF card.

Tune Ollama to 90%

Side-by-side default / suggested / current view of OLLAMA_FLASH_ATTENTION, OLLAMA_KV_CACHE_TYPE, OLLAMA_KEEP_ALIVE, etc. Persists via setx · launchctl · systemd depending on OS.

Themes & i18n

Seven built-in themes (dark, light, Dracula, Solarized, Nord, Monokai) and translations for 20 languages including English, Spanish, Chinese, Hindi, Arabic, Portuguese, Bengali, Russian, Japanese, German, French, Korean, and more.

Headless commands

--view, --list, --backup, --restore, --tune, --launch, --delete let you script every TUI capability from CI, cron, or an editor plugin.

Cross-OS, free, private

Windows · macOS · Linux. Talks only to the public Hugging Face API and your local Ollama daemon. No telemetry, no accounts.

Install

Needs Node.js 20+. Ollama is optional — hfo can install it for you via winget / brew / the official script.

npm i -g hfo-cli

# Open the TUI
hfo

# Pre-fill the Install tab with a model
hfo bartowski/Llama-3.2-3B-Instruct-GGUF

Privacy & cost

hfo makes exactly two kinds of external requests — both to Hugging Face's public, unauthenticated API:

GET https://huggingface.co/api/models/<repo>/tree/main
GET https://huggingface.co/<repo>/resolve/main/<file>

Everything else — nvidia-smi, systeminformation, ollama ps, ollama create, ollama launch — runs locally. No telemetry. No accounts. No external calls beyond those two endpoints and (optionally, on your explicit request) the Ollama install script. Gated or private repos use a standard HF_TOKEN environment variable or --token flag.
